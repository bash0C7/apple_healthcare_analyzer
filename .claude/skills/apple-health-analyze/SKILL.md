# apple-health-analyze スキル

Apple Healthデータを分析し、健康状態・トレンド・フォーキャスト・疑似身体スコアを日本語で提示するスキル。

---

## 使い方

プロジェクトディレクトリ（`/Users/bash/Downloads/apple_health_export`）から以下を実行してJSONサマリーを取得する。

```bash
# デフォルト（直近90日）
bundle exec ruby scripts/analyze.rb

# 直近N日を指定
bundle exec ruby scripts/analyze.rb 30

# 日付範囲を指定（YYYYMMDD YYYYMMDD）
bundle exec ruby scripts/analyze.rb 20260101 20260330
```

取得したJSONには以下の主要フィールドが含まれる（身体スコア算出に使用）:
- `hrv`: HRV（SDNN、ms）— 心拍変動
- `asleep_hours`: 実睡眠時間（InBed除く: AsleepCore + AsleepREM + AsleepDeep）
- `respiratory_rate`: 睡眠時呼吸数（回/分）
- `resting_heart_rate`: 安静時心拍数（bpm）

取得したJSONを元に、以下の4ステップで分析を行い日本語で報告する。

---

## 分析フレームワーク

### ステップ1：現状分析（Health Status）

各指標を期間平均値と比較し、改善・安定・悪化を判定する。

- **step_count**: `recent_7d_mean` vs `mean` — 直近7日が平均を下回れば活動低下
- **body_mass**: `latest` vs `mean` — 増減傾向をコメント
- **resting_heart_rate**: `latest` vs `mean` — 安静時心拍は低いほど良好
- **active_energy**: `recent_7d_mean` vs `mean` — 直近7日の消費カロリー傾向
- **sleep_hours**: `recent_7d_mean` vs `mean` — 直近7日の睡眠時間傾向
- **body_fat**: `latest` — 体脂肪率の現状
- **vo2max**: `latest` — 有酸素能力の現状

**異常値フラグ基準**:
- 任意の指標が平均±2SD を超える場合はフラグを立てる
- `recent_7d_mean` と `mean` の乖離が10%を超える場合は要注目として記載

**重要指標**: VO2Maxの低下 + 安静時心拍の上昇 = フィットネス後退シグナル。この組み合わせは必ずハイライトする。

### ステップ2：身体スコア推移

直近14日分の `daily[].body_battery` を時系列で提示する。

**スコア解釈**:
| スコア | 状態 |
|--------|------|
| 80〜100 | 優秀（Excellent） |
| 60〜79 | 良好（Good） |
| 40〜59 | 普通（Fair） |
| 0〜39 | 低下（Poor） |

パターン分析:
- 平日 vs 週末の差異
- 活動量と睡眠の相関（高活動日の翌日に身体スコアが回復しているか）
- 連続低下・連続上昇の有無

### ステップ3：フォーキャスト（マルチホライズン）

直近30日の`daily`データから線形トレンドを読み取り、4つのホライズンで予測する。

**予測ホライズン:**
| ホライズン | 週数 | 性格 |
|-----------|------|------|
| 1週間後 | 1週 | 現実的な近未来 |
| 1ヶ月後 | 4週 | 習慣の成果が見える頃 |
| 3ヶ月後 | 13週 | 体組成・有酸素能力の変化が現れる頃 |
| 6ヶ月後 | 26週 | 夢のシナリオ |

分析対象: **body_mass / body_fat / step_count / resting_heart_rate / vo2max / hrv**

各指標の予測値 = 最新値 + (週次変化量 × 週数)

表形式で提示:
```
| 指標 | 現在値 | 1週間後 | 1ヶ月後 | 3ヶ月後 | 6ヶ月後 |
```

- トレンドが弱い（変化量が小さい）指標は「横ばい」として記載
- 3ヶ月以降は「線形トレンドが続いた場合の推定。実際は習慣変化・体の適応により変動する」と一言添える
- ポジティブな変化は励ましのコメントを付ける（「このペースで続ければ6ヶ月後にはVO2Max 42.0 → マラソン完走圏内」など）

### ステップ4：アクション提案

データに基づく具体的な提案を1〜3つ。汎用的なアドバイスは不可。データから直接導出すること。

例:
- 「睡眠が6.2時間平均に低下しており、身体スコアスコアとの相関が強い。まず就寝時刻を30分前倒しすることを推奨。」
- 「VO2Maxが直近30日で43.1→41.8と低下。週2回以上の有酸素運動（心拍数130〜150bpm、20分以上）を追加すること。」

## 身体スコア の計算式

身体スコア (0-100) = 3成分の加重スコア（睡眠時間は除外 — Apple Healthの睡眠分析で別途確認）

| 成分 | 配点 | 計算 |
|------|------|------|
| HRV（SDNN） | 55点 | `clamp(hrv / baseline_hrv_mean * 55, 0, 55)` |
| 安静時心拍 | 30点 | `clamp(30 - (rhr - baseline_rhr_p10) * 1.875, 0, 30)` |
| 睡眠時呼吸数 | 15点 | `clamp(15 - |resp_rate - 14.0| * 2.25, 0, 15)` |

- `baseline_hrv_mean`: データセット全体のHRV平均値
- `baseline_rhr_p10`: データセット全体のRHR 10パーセンタイル
- 呼吸数最適値: 14回/分（睡眠中）
- スコア解釈: 80-100=優秀, 60-79=良好, 40-59=普通, 40未満=要注意

## フォーキャスト手法

`daily`配列の直近30エントリを使い、線形回帰の概念で傾向を算出する。

1. 30日分のデータを日付順に並べる
2. 前半15日の平均と後半15日の平均を比較し、方向（増加/減少/横ばい）と変化量を算出
3. 変化量を週次換算して「週あたりXの変化」と表現
4. 各ホライズンの推定値 = 最新値 + (週次変化量 × 週数)
   - 1週間後: × 1
   - 1ヶ月後: × 4
   - 3ヶ月後: × 13
   - 6ヶ月後: × 26
5. トレンドが弱い（変化量が小さい）場合は「横ばい」として報告
6. 長期ほど不確実性が高まることを注記する

## 出力フォーマット

以下のセクション構成で日本語で報告する。

```
## 健康状態サマリー（現状分析）

[分析期間・主要指標の現状・異常値フラグ]

## 身体スコア 推移（直近14日）

[日付 | スコア | 状態 の表形式、またはテキスト列挙]
[パターンのコメント]

## フォーキャスト（4週間後の予測）

[体重・体脂肪・歩数・睡眠・VO2Max等の方向と予測値]
[リスクシグナルがあれば強調]

## アドバイス

1. [具体的な提案]
2. [具体的な提案]
（最大3つ）
```

数値は小数点1桁で揃える。単位を必ず付ける（kg, bpm, 時間, kcal, mL/kg/min）。

---

## 注意事項

- **これは医療アドバイスではありません。** 本スキルによる分析は健康管理の参考情報であり、医療診断・治療の代替となるものではありません。体調に関する懸念がある場合は医師に相談してください。
- データが存在しない指標（`null` や空値）はスキップし、「データなし」と明記する。
- `daily`配列が14日未満の場合は身体スコア推移を取得可能な日数分で対応する。
- `daily`配列が30日未満の場合はフォーキャストの精度が低下する旨を一言添える。
